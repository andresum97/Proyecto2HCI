Preguntas Generales

¿Cómo ingreso sesión en el sistema?

La cuenta que utilizará para obtener acceso a dicha plataforma es su cuenta de administrador de la Universidad del Valle de Guatemala El único requerimiento es tener acceso directo a la plataforma. Si desea darle acceso a otro personal deberá de contactar directamente a soporte técnico. En esta plataforma solo se permite el ingreso de usuarios de la Universidad del Valle de Guatemala tipo administrador.

¿Que información obtengo en esta plataforma?

Podrá obtener la cantidad total de estudiantes que ingresan a la biblioteca de la Universidad del Valle de Guatemala Campus Central. La plataforma muestra la información que recopila el sensor de movimiento que se encuentra inslatado en la entrada principal.

¿Cómo se el estado del sensor?

El sensor se encuentra realizando lecturas constantes en la entrada principal. Esta calibrado de tal forma que cuando detecta una entrada en el espacio de la puerta, entonces se activa. En la sección de encontrará los gráficos que le indicarán el estado del mismo y las lecturas que se encuentra realizando.
Conexión directa al servidor

¿Cómo saber si la conexión al servidor es estable?

El servidor es el encargado de almacenar la información que recopila el sensror, por lo que debe de estar siempre conectado para evitar pérdida de información. Hemos preparado una sección en donde usted podrá revisar si está conectado y transmitiendo información, pues es el encargado de indicarle al administrador el estado del servidor y la conexión. Debe de ingresar a Control de dispositivo y dirigirse al gráfico de Estado del server si la gráfica no muestra actualizaciones contacte a sorpote técnico.

¿Cómo el sensor obtiene la información y como es trasladada al sistema?

A continuación se enlístan los pasos que realiza el sensor para recopilar la información y actualizar el servidor:
  • El sensor está conectado a un dispositivo Arduino
  • El Arduino es el encargado de interpretar la información que recopila el sensor
  • Si el Arduino no está transmitiendo información debe de desconectarlo y volverlo a conectar
  • Al obtener la información el sistema se encarga de realizar la conexión con el servidor.
  • El servidor tiene conexión directa en la plataforma por lo que al momento de obtener un dato automáticamente se actualizará.

¿Qué cantidad de lecturas puede realizar el sensor?

Puede realizar lecturas indefinidas a lo largo del día, está diseñado de tal forma que pueda recopilar información durante todo el día y así tener la cantidad exacta de estudiantes que ingresan a la biblioteca en promedio de una hora, un día o una semana.

¿Qué acción debo de tomar con el servidor y el sistema al finalizar el día?

El servidor dejará de recopilar información a partir de las 22:00 hrs. Ya que a esa hora se realizará el mantenimiento. Al finalizar el dia el dispositivo Arduino se desconectará a las 20:00 hrs.
    Para validar que este comportamiento ocurre debe de seguir los siguientes pasos:
  • Verificar que no hayan lecturas entrantes en el sistema
  • Verificar en Control del dispositivoque no hayan lecturas en los gráficos
